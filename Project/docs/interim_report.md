## 1. 프로젝트 개요

### 1.1 핵심 문제

이커머스 판매자(특히 신용 이력이 짧은 씬파일러)의 금융 소외 문제를 **매출 연동 동적 상환(Revenue-Based Financing, RBF)** 으로 해결한다. 기존 RBF의 한계인 **"과거 매출 기반 1회 결정 + 고정 비율 상환"** 구조를 극복하기 위해 시계열 예측 모델로 미래 매출 분포를 산출하고, **CVaR 최적화 + 강화학습**으로 대출자-셀러 trade-off를 최적화하는 AI 프레임워크를 제안한다.

### 4.1 가계 보호 설계 (사용자 핵심 요구사항)

> 사용자 제기: "판매자들이 실제 생활에서 생활하는 현금 흐름까지 침해하지 않는 수준에서 회수율을 가져가야 한다"

**2-tier burden 모델** ([design_household_protection.md](Project/docs/design_household_protection.md)):

```
매출 (R_t)
  │
  ├── 운전자금 (1 − m_i) × R_t          ← Tier 1: 사업 차원 (매출 비례)
  │
  └── 영업이익 (m_i × R_t)
        │
        ├── 가계 생활비 (L_personal)     ← Tier 2: 개인 차원 (거의 고정)
        │
        └── RBF 가용 = m_i × R_t − L_personal
```

### 4.5 CVaR 정적 최적화 (CVXPY)

#### 4.5.1 CVaR 개념과 도입 필요성

**CVaR (Conditional Value at Risk)** 는 금융·보험 분야에서 표준적으로 사용되는 **꼬리 위험(tail risk) 측정 도구**이다.

```
VaR_α(X) = "최악 α% 시나리오에서의 손실 임계값"
          (예: α=5% → 95% 시나리오는 이보다 작은 손실)

CVaR_α(X) = "최악 α% 시나리오의 평균 손실" (VaR 이상의 평균)
           (예: α=5% → 최악 5% 시나리오의 burden 평균)
```

**왜 평균 burden 대신 CVaR을 사용하는가?**

| 측정 방식 | 약점 |
|---|---|
| 평균 burden 최소화 | "평균은 좋지만 극단적 시나리오에서 큰 침해 발생" 가능 |
| **CVaR 최소화** | **최악 시나리오까지 보호 보장** — 셀러 보호 본질에 부합 |

본 연구는 **셀러 가계 침범의 꼬리 위험을 통제**하는 것이 목적이므로 CVaR이 자연스러운 선택이다.

#### 4.5.2 최적화 수식과 각 항의 의미

**목적함수**:

```
max  E[recovery_capped(r)] − λ × CVaR_α(burden_per_scenario)
 r
s.t. r ∈ [r_min, r_max]  = [0.03, 0.25]
```

각 항의 정확한 의미:

| 항 | 의미 | 본 연구 값 |
|---|---|---|
| **E[recovery_capped(r)]** | 평균 회수액 (target 도달 시 더 회수하지 않음으로 cap) | min(r × ΣR_t, target) |
| **CVaR_α(burden)** | 최악 5% 시나리오의 평균 가계 침범 | α = 0.05 |
| **λ** | 회수 vs 위험 trade-off 가중치 | λ = 1.0 (동등 가중) |
| **r** | 결정 변수 (셀러별 단일 수수료율) | [0.03, 0.25] |

→ **"회수는 최대화 + 가계 위험은 최소화"의 균형점 r 찾기**

#### 4.5.3 Monte Carlo 시나리오 생성

셀러의 미래 매출은 불확실하므로, **각 셀러마다 1,000개의 가능한 매출 경로**를 가상으로 생성하여 모든 경우의 위험을 평가한다.

**생성 방식**:

```
for 각 셀러:
    실제 24개월 매출 시계열 → base path
    
    for k = 1 to 1000:
        시나리오 k = base path × (1 + Gaussian noise)
        noise 강도 = 셀러의 CV (Coefficient of Variation) × 0.3
        # CV가 큰 셀러는 시나리오 변동성도 크게 모델링
        
    → 1,000개 시나리오 행렬 (1000 × 24)
```

**의미**:
- CV가 큰(변동 큰) 셀러: 시나리오 분포 넓음 → 최악 시나리오도 극단적
- CV가 작은(안정적) 셀러: 시나리오 분포 좁음 → 위험 예측 안정

#### 4.5.4 CVXPY 구현 흐름

본 연구는 **CVXPY (Python의 convex 최적화 라이브러리)** 를 사용하여 셀러별로 최적화 문제를 해결한다.

```
[셀러별 최적화 흐름]

Step 1: 시나리오 행렬 생성 (1000 × 24)

Step 2: CVXPY 변수 정의
   - r: 결정 변수 (수수료율)
   - z: VaR 보조 변수
   - xi: CVaR 슬랙 변수 (1000개)

Step 3: 시나리오별 burden 계산
   for k in 1..1000:
       payment[k] = r × scenarios[k]
       burden[k] = max(0, payment[k] − safe_rbf_cap[k])
   
Step 4: CVaR 계산 (Rockafellar-Uryasev 공식)
   CVaR_α = z + (1 / (1 − α)) × mean(xi)
   subject to: xi[k] ≥ burden[k] − z

Step 5: 목적함수 + 제약 결합
   maximize E[recovery_capped(r)] − λ × CVaR_α
   subject to: r ∈ [r_min, r_max]
              xi ≥ 0

Step 6: CLARABEL 솔버로 해결
   → 최적 r* 산출
```

**구현 결과**:
- 1,302 셀러 × 약 0.1초 / 셀러 ≈ 2분 처리
- **1,302 셀러 100% 최적화 성공** (수치 안정성 확보)

#### 4.5.5 결과 — 셀러 위험 프로파일별 r* 차별화

CVaR 최적화의 가장 큰 가치는 **각 셀러의 위험 프로파일에 맞춘 차별화된 r***을 산출한다는 점이다.

**유형별 r* 분포** (m_i = 0.10 적용 시):

| 셀러 유형 | r* 평균 | 해석 |
|---|---|---|
| **growth** (성장형) | **0.059** | 매출 추세 명확 → 미래 회수 자신 → 약간 공격적 |
| **stable** (안정형) | 0.041 | 안정적이지만 영업이익 < 가계비 → 보수적 |
| **other** (기타) | 0.037 | 패턴 모호 → 보수적 |
| **volatile** (불안정형) | 0.037 | 변동 큼 → 위험 회피 → 보수적 |
| **seasonal** (계절형) | 0.035 | 계절 매출 폭락 위험 → 매우 보수적 |
| **decline** (쇠퇴형) | **0.034** | 매출 폭락 위험 → r_min 근처 (최대 보수) |
| **전체 평균** | **0.042** | 가계 보호 우선 학습 결과 |

**핵심 발견**:
- CVaR이 셀러별로 r을 **0.034 ~ 0.059 범위로 차별화** (Fixed 단일 정책과 명확히 구분)
- 위험 프로파일이 위험할수록 r 낮춤 (decline → r_min)
- 매출 추세 좋을수록 r 높임 (growth → 평균보다 위)

→ **이는 단순 Fixed 정책이 불가능한 "셀러별 맞춤 정책"의 첫 단계**

#### 4.5.6 정적 r의 한계 — RL이 필요한 이유

CVaR이 셀러별 차별화는 가능하지만, **하나의 셀러에 대해 24개월 동안 같은 r을 적용**하는 정적 정책이다. 이로 인해 본질적 한계 발생:

```
[CVaR 정적 r = 0.042의 결과]

매출 큰 달 (R = 1500만):
  payment = 1500 × 0.042 = 63만
  → 회수 기회 큰데 작게 가져감 (손해)

매출 작은 달 (R = 200만):
  payment = 200 × 0.042 = 8.4만
  → 가계비 충당은 됐지만 회수도 미미

24개월 평균:
  recovery = 0.28 (Fixed-0.15의 1.000 대비 28%)
  → 매우 보수적 → 회수 실패
  → completion 0% (모든 셀러 미완납)
```

**근본 원인**: **매월 매출이 다른데 r은 고정**.

| 상황 | 이상적 r | CVaR 정적 r (0.042) |
|---|---|---|
| 매출 큰 달 | r 증가 (회수 가속) | 0.042 (작음, 기회 손실) |
| 매출 작은 달 | r 감소 (가계 보호) | 0.042 (적당) |
| 누적 회수 부족 | r 증가 | 0.042 (변화 없음) |
| 누적 회수 충분 | r 감소 | 0.042 (변화 없음) |

→ **한 가지 r 값으로 모든 월·모든 상황 대응 불가능**

#### 4.5.7 CVaR의 학술적 위치

CVaR은 본 연구에서 **두 가지 역할**을 수행:

1. **독립 정책으로서의 비교 baseline**: "가계 보호 우선" 정적 정책의 베스트 — 회수 손해 정량화
2. **RL 정책의 학술적 기준**: PPO가 "CVaR보다 가계 보호 + Fixed보다 회수" 동시 달성해야 의미 입증

→ 다음 절(4.6 PPO RL)에서 **동적 r 조정**으로 CVaR의 한계 극복을 검증한다.

### 4.7 핵심 발견 — RL의 강점과 한계 (Lv4 정정)

**시뮬레이션 환경에서 관찰된 PPO 우위**:
1. **Completion 92.4%** (전체 1,302명 평가) — 동시에 회수율 최고. train-test 격차 +1.6%p (overfit 없음)
2. **HH 침범 월수 12.2** — Fixed/CVaR(20.7) 대비 41% 감소 (침범 빈도 측면)
3. **가계 안전 셀러 비율** Fixed/CVaR 0% → PPO 2.7% (극히 적지만 유일하게 일부 셀러 보호)

**침범 정도까지 고려한 정직한 평가 (A-2 신규 발견)**:

| 정책 | 침범 월수 | 누적 침범액 | 단일월 최대비 | 상위 5% 셀러 최대비 |
|---|---|---|---|---|
| Fixed-0.15 | 20.7 | 2,209 만 | 1.26 | 2.52 |
| **CVaR** | 20.7 | **689 만** ✅ | **0.32** ✅ | **0.99** ✅ |
| **PPO** | **12.2** ✅ | 1,834 만 | **1.73** ❌ | **3.60** ❌ |

- **PPO는 침범 빈도(월수)는 줄였으나, 침범 발생 시 정도는 가장 큼**
- 상위 5% 셀러에서 PPO는 단일 월 가계비의 **3.6배까지 침범** (CVaR 1배 수준 대비 매우 큼)
- 즉 "**침범 빈도 ↓ + 침범 강도 ↑**" 정책 학습 — 종합 평가가 단순하지 않음

**PPO 변형 시도 (Lv4) — 4가지 시도 모두 한계**:
- 시계열 분포 통합 PPO (lag-based): Completion +0.8%p, 침범 측면 변화 미미
- Reward 강화 PPO (η=5x, binary): Completion +0.4%p, 침범 정도 변화 없음
- **Reward 비례 PPO (X1: η × violation_ratio)**: 침범 강도 1.73 → 0.21 ✅, **Completion 0% ❌**
- **Reward 비례 + r_clipping PPO (X1+X2)**: 침범 강도 0.20 ✅, **Completion 0% ❌**
- → **결정적 발견**: 세 가지 (회수율 + 침범 빈도 + 침범 강도) **동시 최적화는 본 합성 데이터 구조상 불가능**

**원인 진단 (학술 가치 큰 결론)**:
- 합성 영세 셀러 대다수가 영업이익(매출×m_i=0.10) < 가계비(128만) 구조
- 이런 셀러는 침범 없이는 RBF 회수 불가능 (수학적 한계)
- Reward shaping이 아닌 **"사전 적합도 평가 (대출 거절)" 로 해결해야 함**
- 즉 모든 셀러를 RBF 대상으로 보면 trade-off 본질적 → 매출 상위 셀러 선별 필요
- 본 보고서 9장 "학습 vs 운영 패러다임"의 **사전 적합도 평가 단계** 정당화

### 4.8 본 연구의 운영 알고리즘 — 셀러별 (L\*, T\*, cap\*) 동적 최적화 + 침해 0% 엄격 거절

**알고리즘 (Day 11+ 결과)**:

```
[Step 1] 셀러 정보 (R, type, cv) 수신

[Step 2] cap*(seller) 동적 산출 (위험 프리미엄)
  cap* = 1.10 + risk_premium(type, cv)
    stable: -0.05 → cap* 1.05 (5% 이자)
    volatile: +0.20 → cap* 1.30 (30% 이자)

[Step 3] L_max(R) 산출 — 침해 0% 보장
  monthly_safe = R × m_i - L_personal
  L_max = T_max × monthly_safe / cap*   (T_max = 36)

[Step 4] 거절 판정
  monthly_safe ≤ 0    → 거절 (가계비 충당 불가)
  L_max < 100만       → 거절 (비즈니스 가치 부족)

[Step 5] 허용 시 (L*, T*, cap*) 적용
  L* = L_max, T* = 36, cap* = 위에서 결정
  → PPO/CVaR/Fixed 중 셀러/플랫폼 선호 정책 적용
```

**1,302명 합성 셀러 적용 결과**:

| 분류 | 셀러 수 | 비율 | 이유 |
|---|---|---|---|
| **거절** | 1,142 | **87.7%** | 가계비 충당 불가 87.4% + 비즈니스 가치 부족 0.3% |
| **적합** | 160 | **12.3%** | stable 122 + growth 38 (변동 셀러 전체 거절) |

**적합 셀러 160명에 5-정책 적용 결과**:

| 정책 | Completion | HH 안전 % | 단일월 최대 침해비 | P95 침해비 | 특징 |
|---|---|---|---|---|---|
| Fixed-0.15 | **100%** | 0% | 2.34 | 4.08 | 회수 우위, 침해 큼 |
| PPO base | 52.5% | 15.6% | 1.81 | 4.02 | 중간 |
| PPO eta_strong | 54.4% | 15.6% | 1.85 | 4.16 | 중간 |
| **PPO x1 비례** | 32.5% | 16.3% | **0.21** ✅ | **0.30** ✅ | 안전 절대 우위 |
| **CVaR** | **86.9%** | 0% | **0.92** | **1.02** | **회수 + 안전 균형** ⭐ |

**핵심 발견**:
1. **(L\*, cap\*) 적용에도 일부 침해 발생** — 평균 매출 R 기반 산출이라 R_t 변동 시 침해 (본질적 한계, 분산 보정 필요)
2. **CVaR이 적합 셀러에서 최강 균형**: 회수 87% + 단일월 최대비 0.92 (가계비 거의 침해 없음)
3. **PPO 동적 r의 가치 제한적**: 정적 CVaR보다 회수율 낮음. 다만 PPO x1 비례는 안전 측면 절대 우위
4. **본 연구의 시스템 가치**: "**한국 영세 셀러 87%는 RBF 거절 + 적합 12% 셀러에게 안전 동적 RBF 제공**"

### 4.9 본 연구의 학술적 차별점 (최종 정리)

**기존 RBF 상품 (Clearco, Pipe, 한국 PG 매출자금 대출)** vs **본 연구**:

| 측면 | 기존 RBF | 본 연구 |
|---|---|---|
| 매출 비례 회수 (P = R × r) | ✅ | ✅ |
| **r 자체 동적 조정 (매월)** | ❌ (r 고정) | ✅ **차별점 1** (PPO) |
| 셀러 신용 평가 | ✅ | ✅ |
| **가계비 보호 강제** | ❌ | ✅ **차별점 2** (2-tier burden) |
| **셀러별 (L\*, T\*, cap\*) 자동 산출** | 부분적 | ✅ **차별점 3** |
| 사전 거절 정량 기준 | 신용/매출만 | ✅ + **가계 안전 조건** |

**본 연구의 unique 위치**:
> **세계 최초의 "동적 r + 가계 안전 강제 + 셀러별 (L\*, T\*, cap\*) 최적화 RBF 시스템"**의 학술적 프로토타입. 한국 영세 셀러 시장의 RBF 적용 가능 비율 (~13%)을 정량 입증하고, 적합 셀러에 대한 다양한 정책 선호별 결과를 제시.

**정책 강점 분화 — 의미 있는 결론**:
- **CVaR**: 가계 보호 절대 우위 (모든 침범 지표 최저). 회수율 0% — 보수적
- **PPO 계열**: 회수율 절대 우위 (92-93%). 침범 정도는 큼 — 공격적
- 두 정책은 **다른 목적함수를 최적화**. 사용자/플랫폼 정책 선호에 따라 선택

> **수정된 핵심 결론**: RL은 정적 정책의 trade-off를 "완전히 해결"하지 못하며, 침범 빈도와 강도 사이의 trade-off는 여전히 존재. 본 연구는 RL이 회수율과 침범 빈도를 동시 개선함을 시뮬레이션 환경에서 관찰. 침범 강도 측면에서는 CVaR이 우수하므로, 실제 RBF 운영에서는 셀러 위험 프로파일별로 두 정책 결합 또는 선택이 합리적.

| Fixed-0.15 | 81.54 ± 0.00 | 0.14 ± 0.02 | 0.25 ± 0.81 | 20.71 ± 0.66 |
| **PPO** | **91.60 ± 0.52** | 0.22 ± 0.01 | **3.18 ± 2.29** | **12.46 ± 0.67** |

**핵심 발견**:
1. **PPO가 모든 metric에서 우위 일관 유지**:
   - HH 침범 개월: 12.46 (Fixed의 60.2%, CVaR의 66.5%)
   - HH 안전 %: 3.18 (Fixed의 12.7배, CVaR의 1.7배)
   - Completion: 91.60% (Fixed의 112%)
2. **PPO의 robustness 가장 강함**:
   - HH 침범 개월 std = 0.67 (CVaR 3.68의 1/5)
   - Completion std = 0.52 (환경 변동에 매우 안정)
3. **Fixed-0.15 std = 0**: 정적 정책이라 환경 변동 무관 (회수율 측면). 단 burden·HH 측면은 환경 영향 받음

### 6.3 Tornado Plot 해석 (변수별 영향력)

각 정책에서 L_personal·m_i 변동이 metric에 미친 영향 폭(range = max - min):

**Completion % 변동 폭**:
- Fixed-0.15, CVaR: 0 (정적 정책 → 환경 변동 무관)
- **PPO: 1.15** (m_i 변동에만 반응, RL state에 m_i 정규화 값 입력됨)

**Mean Burden 변동 폭**:
- 모든 정책 비슷 (~0.01-0.03)
- L_personal·m_i 둘 다 비슷한 영향

**HH 안전 셀러 % 변동 폭**:
- Fixed-0.15: L 0 / m_i 0.77 (정적 정책)
- CVaR: L 5.38 / m_i 5.36
- **PPO: L 5.77 / m_i 5.77** (변동 큼이지만 절대값도 가장 큼)

**평균 HH 침범 개월 변동 폭** ⭐ (본 연구 핵심 metric):
- Fixed-0.15: 0 (정적)
- CVaR: **L 1.62 / m_i 1.13**
- **PPO: L 1.75 / m_i 1.08** (CVaR과 비슷한 변동 폭, 단 평균은 PPO가 가장 낮음)

→ **PPO는 HH 침범 개월에서 평균 우위 + 변동 폭 작음** = robust한 가계 보호 정책

### 6.4 학술적 결론

> **"본 연구의 PPO 정책은 m_i ∈ [0.05, 0.20], L_personal_min ∈ [64, 192]만원의 42개 가정 조합에서 평균 HH 침범 개월 12.46 ± 0.67(정적 정책의 약 60% 수준)을 일관되게 유지하였다. HH 안전 셀러 비율은 평균 3.18%로 Fixed-0.15(0.25%) 및 CVaR(1.91%) 대비 명확한 우위를 보였다. 변동성(σ)도 정적 정책 수준(0.67 vs Fixed 0.66)으로 가정 정확성에 의존하지 않는 robust 결과임이 입증되었다.**
>
> **이는 본 연구의 모델이 한국 이커머스 셀러 영업이익률 공식 통계 부재 환경에서도 일반화 가능한 정책을 학습함을 의미하며, 실제 RBF 운영 시 셀러별 정확한 m_i·매출 정보가 입력될 때 본 모델의 정책 가치가 더욱 강화될 것임을 시사한다."

## 7. 학술적 기여 및 차별점

| 기존 RBF 연구 | 본 연구 |
|---|---|
| 회수율·디폴트율만 최적화 | + **셀러 가계 생활 보호** (2-tier burden) |
| 단일 burden 지표 | **2-tier burden** (사업/가계 분리) |

### 본 연구의 정량적 발견 (Lv4 정정)

1. **PPO 회수율 우위 + 침범 빈도 감소**: Completion 92.4% (Fixed 79.7%), HH 침범 월수 12.2 (vs 20.7) — 전체 1,302명 평가, train-test 격차 1.6%p (robust)
2. **단, 침범 정도(액도) 측면에서는 CVaR이 우수**:
   - CVaR 단일월 최대 침범비 0.32 / 상위 5% 셀러 0.99 (가계비 1배 수준)
   - PPO 단일월 최대 1.73 / 상위 5% 셀러 3.60 (가계비의 3.6배까지)
   - → "침범 빈도 ↓ + 침범 강도 ↑" 정책 학습
3. **PPO 변형 시도 효과 제한적**: 분포 통합 (+0.8%p completion), Reward η 5x 강화 (+0.4%p completion). 침범 강도 측면 변화 없음
4. **민감도 robust**: m_i, L_personal ±50% 변동에서도 정책 우열 일관 (42 조합)
5. **한국 이커머스 영세 셀러는 영업이익만으로 가계비 충당 어려운 구조 정량화** — RBF는 매출 상위 셀러(연 1억+, 약 8%)에 적합
6. **정책 강점 분화 — 사용자/플랫폼 선택 필요**: CVaR (가계 보호) vs PPO (회수율) 둘 다 강점/약점 명확

## 8. 정직한 한계 인정 (Lv4 강화)

본 보고서는 **사실 확인 기반**으로 작성되며, 학술적 약점도 명시한다.

1. **합성 데이터 의존**: 한국 셀러 실데이터 부재로 합성 코호트 사용. KOSIS r=0.55로 시장 패턴 반영하나 generative process artifact 가능성 (한국 시장 본질이 아닌 "Olist + KOSIS 혼합 구조" 일부 반영)
2. **Olist 클러스터링 약점**: Silhouette 0.25, 4유형 자연 분리 어려움 → 분산 파라미터 추출용으로만 사용
3. **시계열 예측 정확도 한계**: WAPE < 20% 셀러 비율 17.8% (v2, 점 예측), Coverage 60.7% (분포 예측, 목표 80%). 단기/장기 horizon 큰 차이 없음. Phase 5에서 더 정교한 분포 통합 필요
4. **PPO 침범 정도 한계 (A-2 발견)**: PPO는 침범 빈도는 줄이나 발생 시 정도는 크다. 단순 reward 가중치 조정(η 5x)으로 해결 안 됨. r_max 제한 또는 침범액 비례 페널티 등 본질 reward 재설계 필요 — 후속 연구
5. **Reward shaping 의존성**: 본 결과는 현재 reward 가중치 (α=1, β=2, γ=5, δ=10, η=1)에 의존. 다른 가중치에서 우열 다를 수 있음. 민감도는 m_i/L_personal에만 검증함
6. **m_i 추정값 한계**: 한국 이커머스 셀러 영업이익률 공식 통계 부재. **m_i = 0.10은 스마트스토어 gross margin 20%에서 운영비 10% 차감 추정**. 한국은행 도·소매 평균 3.4%와 격차 큼 — 영세 셀러는 더 낮을 가능성. Phase 4 민감도 분석으로 0.05~0.20 robustness 검증
7. **L_personal 가구 다양성 단순화**: 1인 자영업자 가정. 가구 크기별 차별화는 한국 셀러 매출-가구 동시 측정 데이터 부재로 후속 연구
8. **카테고리별 m_i 차별화 부재**: 도·소매 카테고리별 영업이익률 분리 통계 부재로 단일값 사용. 학습 vs 운영 패러다임에서 운영 단계에 보완 가능
9. **외생 변수 영향 강도 가정**: KOSIS-Naver 실측 상관 |r|=0.226 측정 후 합성 generative에 반영 시도 (cohort_kr_v6, β=0.4). 그러나 시계열 정확도 개선 효과 미미 → 24개월 짧은 시계열의 본질적 학습 한계
10. **결과 일반화 한계 — 가장 중요**: 본 결과는 합성 데이터 기반 시뮬레이션이며, 실제 한국 이커머스 셀러 데이터로 검증되지 않음. "이론적 가능성 제시" 수준이며 실제 운영 적용 전 실증 검증 필수
